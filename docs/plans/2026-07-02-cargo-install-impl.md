# Cargo install (cargo-binstall bootstrap + Rust packages rule) â€” Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap `cargo-binstall` at build time as infra, install `topgrade` as the one build-time cargo tool via `cargo binstall --only-signed -y`, declare 5 runtime-manual cargo tools at a new `layer = 6`, and write the Rust packages rule (spec 24) â€” with `packages.toml` the single hand-edited SoT and the image secret-free.

**Architecture:** `cargo-binstall` is curl-bootstrapped in the Containerfile `toolchain` stage from a version-pinned (v1.20.1) + SHA256-verified prebuilt tarball (infra, NOT in `packages.toml` â€” same category as `rustup`/`mise`). Build-time cargo tools (`layer = 3`, currently `topgrade`) install via `cargo binstall --only-signed -y`. Runtime-manual cargo tools (`layer = 6`) are declared in `packages.toml` for SoT but NOT build-installed â€” the generator emits `layer_6/cargo.txt` as a reference list the Containerfile never reads. The `toolchain` sub-layers reorder to 3-2 rustup / 3-3 mise binary / 3-4 mise languages / 3-5 cargo-binstall bootstrap / 3-6 cargo tools.

**Tech Stack:** Python 3.14 + `tomllib` (codegen `programs/generate_deps/main.py`), pytest (tests), Bash + zsh + Podman/BuildKit (Containerfile), cargo-binstall (prebuilt binary installer), cargo (rustup-provided).

## Global Constraints

- `cargo-binstall` is pinned to **v1.20.1**, asset `cargo-binstall-x86_64-unknown-linux-musl.tgz`, SHA256 **`f12954bc382e1d0b2df3fbfb217a05d92c25570e4517841e0613499a24f4594e`** (verified by download + `sha256sum`). The tarball is single-file (`tar -tzf` lists only `cargo-binstall`). An upgrade is a deliberate edit of the `ARG` + SHA literal.
- The build-time cargo install flag set is **`cargo binstall --only-signed -y`**. `--secure` is NOT a real flag (the review was wrong); `--only-signed` is the correct strict flag (reject unsigned-only). `-y` = `--no-confirm` (alias). Verified `topgrade` resolves via a signed source (QuickInstall, sigstore-verified) so `--only-signed` succeeds.
- `dependencies/packages.toml` is the **only hand-edited** package manifest (I-FS3); every `dependencies/layer_*/` file is generated by `make gen-deps` and must never be hand-edited.
- Each `[[tool]]` entry requires `name` (str), `manager` (enum), `layer` (int â‰Ą 0), `has_configs` (bool); `description` (str, optional). `schema` stays at **1**.
- `layer = 6` = **runtime-manual reference list** (no Containerfile stage; build only reads `layer_3`). The generator emits `layer_<N>/<manager>.txt` for any `layer >= 1` with entries â€” no functional emission change; only a `render_doc_block` heading special-case.
- Containerfile `RUN` shells are `zsh -c`; word-splitting uses `${=pkgs}` (zsh split operator), **not** `$pkgs`.
- Every `toolchain`-stage `RUN` sources `/tmp/build-home/.zshenv` first (materializes `CARGO_HOME`/`RUSTUP_HOME` and puts `$CARGO_HOME/bin` on PATH).
- **No BuildKit cache mount for binstall downloads** (E-F1: `cargo-binstall` has no persistent download cache â€” per-run RAII `tempdir` in `$CARGO_HOME`; verified). The `cargo/registry`+`cargo/git` cache mounts are dropped from Layer 3-6 (binstall uses the crates.io HTTP API, not the on-disk index) â€” they remain on Layers 4-1/4-2 for `paru`.
- **Rollout (E-F2):** an existing `dotfiles_cargo` named volume will NOT pick up the new `$CARGO_HOME/bin` binaries (`cargo-binstall`, `topgrade`) on `make up`; run `make clean` (or `podman volume rm dotfiles_cargo`) before the first `make up` after this change.
- No secret is baked into any image layer (extends I4 / spec 13 I-S4 â€” public binaries over HTTPS, no credentials).
- Commits: one commit per Phase. Commit message references the issue path `docs/issues/2026-07-02-cargo-install.md`.

"""Tests for the layer-6 runtime-manual reference list in generate_deps.

from pathlib import Path

def test_render_doc_block_layer6_heading() -> None:
    tools = [{"name": "maturin", "manager": "cargo", "layer": 6,

              "has_configs": True}]

    assert "#### Layer 0 â€” already in the base image" in block


def test_write_txt_files_emits_layer_6_cargo_txt(tmp_path, monkeypatch) -> None:
    monkeypatch.setattr(main, "DEPS_DIR", tmp_path)
    by_layer = {
        6: [{"name": "maturin", "manager": "cargo", "layer": 6,
             "has_configs": False}],
    }
    n = main.write_txt_files(by_layer)
    out = tmp_path / "layer_6" / "cargo.txt"
    assert n == 1
    assert out.is_file()
    assert "maturin" in out.read_text()


def test_layer6_entries_do_not_leak_into_layer3(tmp_path, monkeypatch) -> None:
    """A layer=6 cargo entry must NOT appear in layer_3/cargo.txt."""
    monkeypatch.setattr(main, "DEPS_DIR", tmp_path)
    monkeypatch.setattr(main, "EXPECTED_EMPTY_FILES", ((3, "cargo"),))
    by_layer = {
        3: [{"name": "topgrade", "manager": "cargo", "layer": 3,
             "has_configs": False}],
        6: [{"name": "maturin", "manager": "cargo", "layer": 6,
             "has_configs": False}],
    }

    l3 = (tmp_path / "layer_3" / "cargo.txt").read_text()
    l6 = (tmp_path / "layer_6" / "cargo.txt").read_text()
    assert "topgrade" in l3 and "maturin" not in l3
    assert "maturin" in l6 and "topgrade" not in l6

# 24 â€” Rust packages rule

> Spec status: **active**. Normative rule for selecting the install
> manager of a package that depends on the Rust toolchain (a Rust crate /
> cargo-installable tool), and for the build-time vs runtime-manual
> boundary. The `layer` field semantics (including `layer = 6`) are in
> [`02-installed-programs.md`](02-installed-programs.md); the build-flow
> stage table is in [`21-container-build-flow.md`](21-container-build-flow.md).

## Â§1 Scope

This rule applies to any package that depends on the Rust toolchain â€”
i.e. is itself a Rust crate or cargo-installable tool. Packages that do
NOT depend on the Rust toolchain use `pacman` / `paru` / `nix` / `uv` as
usual and are out of scope here.

## Â§2 Manager selection

- **`paru` (AUR prebuilt)** â€” preferred when ALL of:
  (a) the package does NOT depend on the rust toolchain (paru packages
  should not pull `rust` / `rustup`), AND
  (b) a stable prebuilt AUR binary exists, AND
  (c) the operator does not need beta / latest-from-source.
- **`cargo-binstall` (prebuilt binary)** â€” preferred when the package IS
  a rust-toolchain tool and ships a prebuilt binary (fast, no compile,
  latest available). Build-time use MUST pass `--only-signed` (Â§3).
- **`cargo install --locked` (source)** â€” only when the package has NO
  prebuilt binary (source-only crate) OR the operator needs beta /
  latest-from-source. Slow (compiles). Runtime-manual only (`layer = 6`).

## Â§3 Build-time vs runtime-manual boundary

- **`layer = 3`** â€” build-installed, via `cargo binstall --only-signed -y`
  (signed prebuilt only). `--only-signed` rejects unsigned-only packages
  (the `--secure` flag does NOT exist in cargo-binstall; `--only-signed`
  is the correct strict flag; `-y` = `--no-confirm`). Use only for tools
  the container must boot with (e.g. `topgrade`, which resolves via a
  signed QuickInstall source). Keep this list small (build cost + image
  size).
- **`layer = 6`** â€” runtime-manual reference list. Declared in
  `packages.toml` for SoT but NOT build-installed; the operator runs
  `cargo binstall <pkg>` (or `cargo install --locked <pkg>` for
  source-only) after `make up`. Use for the bulk of wanted cargo tools,
  especially beta / latest / source-driven ones.
- **Failure mode + recovery:** if a `layer = 3` cargo entry has no
  signed prebuilt source, `cargo binstall --only-signed -y` fails at
  Layer 3-6 with a "no signed artifact found" error. Recovery: move the
  entry to `layer = 6` (runtime-manual; install via `cargo install
  --locked` or `cargo binstall` without `--only-signed` at the
  operator's discretion). The generator cannot pre-validate
  signed-prebuilt availability (no network at gen time); this rule is
  the only guard.

## Â§4 `cargo-binstall` is infrastructure

`cargo-binstall` itself is **infrastructure** (Containerfile bootstrap,
NOT a `packages.toml` entry), extending the curl-bootstrap precedent of
`rustup` (spec 21 row 3-2) / `mise` (row 3-3), now formalized as
[`20-container-rules.md`](20-container-rules.md) `I-INFRA1` with
`I-CARGO1` as its cargo instance. The bootstrap is version-pinned +
SHA256-gated (currently v1.20.1, SHA256
`f12954bc382e1d0b2df3fbfb217a05d92c25570e4517841e0613499a24f4594e`); the
pinned version + SHA are `ARG`s in the Containerfile so an upgrade is a
deliberate, auditable edit. binstall invocations are public-binary-only
(no authenticated-source binstall shares a cache â€” there is no binstall
cache mount; see spec 21 / `I-CARGO1`).

## Â§5 `layer = 6` convention

No Containerfile stage 6 exists (build stages are 0-5). `layer = 6` is a
reference list only: the generator emits `layer_6/<manager>.txt`
automatically for any `layer = 6` entries (no functional generator
change); it is NOT in `EXPECTED_EMPTY_FILES`, so an empty layer-6 set
emits no file. The Containerfile never `COPY`s `layer_6/*`. The spec 02
AUTO-GEN block renders the heading "Layer 6 â€” runtime-manual (not
build-installed)".
