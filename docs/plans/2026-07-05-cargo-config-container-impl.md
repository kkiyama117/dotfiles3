**Goal:** Make `dot_local/share/cargo/config.toml` a normal chezmoi-managed Cargo config at runtime and copy the rendered config into the container build before Rust dependency work.

**Architecture:** Use the existing build-prepass render path: `.chezmoiignore` re-includes only `$CARGO_HOME/config.toml`, Stage 2 renders it under `/tmp/build-home`, and an early Stage 3 sub-layer copies that rendered file into the live `$CARGO_HOME`. Generated dependency outputs stay derived from `dependencies/packages.toml`, while hand-written specs document the new Cargo config invariant and build-flow ordering.

**Tech Stack:** Chezmoi source layout, `.chezmoiignore`, Manjaro `Containerfile`, zsh build `RUN` snippets, Podman/BuildKit, `make gen-deps`, Python dependency generator tests, Markdown specs.

## Global Constraints

- `dot_local/share/cargo/config.toml` maps to `$CARGO_HOME/config.toml`; `dot_zshenv.tmpl` sets `CARGO_HOME=$XDG_DATA_HOME/cargo`.
- Runtime `chezmoi apply` manages only `~/.local/share/cargo/config.toml` under `$CARGO_HOME`; cargo binaries, registry data, and git cache remain volume-owned and unmanaged.
- The container build must copy the rendered `/tmp/build-home/.local/share/cargo/config.toml` to `$CARGO_HOME/config.toml` before any Rust, cargo, or AUR step can compile Rust code.
- The Containerfile must consume the rendered file from `/tmp/build-home`, not the raw chezmoi source path.
- `dependencies/packages.toml` is the only hand-edited package manifest; every `dependencies/layer_*/` file and the `02-installed-programs.md` AUTO-GEN block are generated by `make gen-deps`.
- The Linux Cargo config requires `clang`, `compiler-rt`, and `mold` from Layer 1 pacman packages.
- No secret material is introduced; the Cargo config contains linker choices only.
- Do not commit unless the user explicitly asks for commits during execution.

- [ ] **Step 2: Update `.chezmoiignore`**

  Replace the current cargo ignore under `# Toolchain / keyring volume mountpoints`:

  ```gitignore

  ```


- [ ] **Step 3: Add the Containerfile cargo config copy sub-layer**

  In `container/Containerfile`, insert this new sub-layer immediately after `FROM build-prepass AS toolchain` and before the current rustup layer:

  ```dockerfile
  # Layer 3-2: Copy rendered Cargo config into $CARGO_HOME before Rust work
  RUN zsh -c 'set -eo pipefail; \
        source /tmp/build-home/.zshenv; \
        cargo_config=/tmp/build-home/.local/share/cargo/config.toml; \
        if [ ! -s "$cargo_config" ]; then \
          echo "toolchain: missing rendered cargo config: $cargo_config" >&2; \
          exit 1; \
        fi; \
        install -d "$CARGO_HOME"; \
        cp "$cargo_config" "$CARGO_HOME/config.toml"; \
      '
  ```

  Expected behavior: the build fails early if the Stage 2 render did not produce the Cargo config.

- [ ] **Step 4: Renumber Stage 3 comments and overview**

  In `container/Containerfile`, update Stage 3 comments so the order reads:

  ```text
  3-2 cargo config copy
  3-3 rustup
  3-4 mise-managed languages
  3-5 cargo-binstall bootstrap
  3-6 cargo tools
  ```

  Also update the Stage 3 overview comment to say it installs Cargo config, rustup, mise, and cargo binaries from the build-prepass scratch render. Keep the existing cache-mount explanation, adjusted to the new sub-layer numbers.

- [ ] **Step 5: Verify the rendered config is no longer ignored**

  Run:

  ```bash
  cd /data/dotfiles3 && chezmoi ignored --source . --path-style absolute | rg '\.local/share/cargo/config\.toml'
  ```

  Expected result: no matching lines. `rg` exits 1 when there are no matches; that is the desired result because `config.toml` must not be ignored.

- [ ] **Step 6: Verify the cargo tree is still protected**

  Expected output includes ignored paths under `.local/share/cargo` if any unmanaged cargo paths exist in the source/destination being checked. If the command prints no cargo paths because only `config.toml` exists in the source tree, confirm by inspection that `.chezmoiignore` still has `.local/share/cargo/*`.

**Acceptance:** `.chezmoiignore` re-includes only `.local/share/cargo/config.toml`; `container/Containerfile` copies `/tmp/build-home/.local/share/cargo/config.toml` to `$CARGO_HOME/config.toml` before Rust/cargo/AUR work; Stage 3 comments match the implemented order.

## Execution Notes

- Existing `dotfiles_cargo` volumes are not repopulated from a rebuilt image. After the `.chezmoiignore` carve-out, runtime `chezmoi apply` should create or update `config.toml` inside the mounted volume. Tests that rely on image-seeded cargo binaries still need the existing volume reset flow (`make clean` or removing `dotfiles_cargo`).
- The plan intentionally does not change `dot_local/share/cargo/config.toml`; it only makes the existing file operational in build and runtime contexts.
- If `make build` is too expensive during implementation, `git diff --check`, `make gen-deps`, `make test-deps`, and a targeted `podman build --target aur` with the same build args/context as `Makefile` are the minimum pre-runtime checks. Full acceptance still requires the repository-authorized `make build` path.
