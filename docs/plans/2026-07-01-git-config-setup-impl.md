# `git config` setup (chezmoi-managed `~/.config/git/{config,ignore}` + `git-delta` + GPG signing) — Implementation Plan

**Status:** pending
**Spec:** [`docs/specifications/implementations/2026-07-01-git-config-setup-design.md`](../specifications/implementations/2026-07-01-git-config-setup-design.md)
**Parent issue:** [`docs/issues/2026-07-01-git-config-setup.md`](../issues/2026-07-01-git-config-setup.md)
**Review trail:** conversational approval 2026-07-01 (brainstorming Q1=B scope, Q2=B `runtime` flag, Q3=A GPG signing; lightweight path — the change introduces a chezmoi data var + a package + two static-ish config files; no new secret transport, and GPG *secret* keys remain in the deferred Bitwarden import).

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring `~/.config/git/config` and `~/.config/git/ignore` under chezmoi management in this repo, wired to `.chezmoidata/git_config.yaml`, so the config is reproducible across host and container — adding `git-delta` (Layer 1) so the `delta` pager works everywhere, gating `credential.helper=libsecret` to host runtime via a new `runtime` chezmoi data var (`DOTFILES_RUNTIME`), and keeping GPG commit signing on in all modes (reusing the gnupg container setup).

**Architecture:** No new build stage. `dot_config/git/config.tmpl` renders the git config from `.chezmoidata/git_config.yaml` identity data, with `credential.helper` gated by `{{ if and (not .build_mode) (eq .runtime "host") }}`. `dot_config/git/ignore` is a static verbatim port of the host global gitignore. `.chezmoi.toml.tmpl` gains a `runtime` data var (default `host`); `entrypoint.sh` exports `DOTFILES_RUNTIME=container`. `git-delta` enters via Layer 1 pacman (`packages.toml`). `commit.gpgsign=true` / `gpg.format=openpgp` / `user.signingkey` render everywhere (signing is NOT runtime-gated).

**Tech Stack:** chezmoi (templates + `.chezmoidata`), Podman/BuildKit, Manjaro base, Arch `extra` `git-delta`, GPG signing (existing `gnupg`/`pinentry` + `dotfiles_gnupg` volume).

## Global Constraints

- **Secret-free image** (spec 13 / spec 20 I4 / I-GIT5): no GPG *secret* key is baked. `user.email` and `user.signingkey` (a public GPG subkey ID) are acceptable plain text (spec 13 §2 Tier 2) and are already public in the committed `.chezmoidata/git_config.yaml`. The GPG secret key lives only in the runtime `dotfiles_gnupg` volume (deferred Bitwarden import).
- **All packages from `packages.toml`** (spec 20 I5/I8): `git-delta` is declared in `dependencies/packages.toml`; `layer_1/pacman.txt` is generated by `make gen-deps`, never hand-edited.
- **No secret-store daemon** (spec 20 I-GPG9): `credential.helper=libsecret` is host-only; the container has no keyring daemon. `libsecret` the library arrives via `pinentry` (gnupg setup); no `gnome-keyring`.
- **`build_mode` is build-time vs runtime, not host vs container.** Both host and container run with `build_mode=false`. The new `runtime` var (`host` | `container`, default `host`) is the host/container signal, driven by `DOTFILES_RUNTIME` (set only in `entrypoint.sh`). The build prepass (`build_mode=true`) auto-omits the gated line — no `Containerfile` change.
- **`USERNAME` defined in `.env`** at repo root (existing). `make build` / `make up` abort when unset.
- **One commit per task** (doc-mgmt §6.5).

> **Plan adjustment vs design §4 staging.** The design staged: 1 data, 2 templates, 3 ignore+toml+entrypoint, 4 packages, 5 specs, 6 smoke. This plan reorders to **1 data → 2 packages → 3 toml+entrypoint+ignore → 4 templates → 5 specs → 6 smoke**, because Task 4's template-render verification needs both the filled `signingkey` (Task 1) and the defined `runtime` var (Task 3) to render without error. Reordering makes each task independently verifiable; no behavior change.
