# mise-Managed Languages (go / python / deno) ŌĆö Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install go, python, deno via mise in the container `toolchain` stage, surfaced at runtime via the existing mise shims, with `packages.toml` as the single hand-edited SoT.

**Architecture:** Make `mise` a list manager in `programs/generate_deps/main.py` (mirroring pacman/paru/cargo) so it emits `dependencies/layer_3/mise.txt` with one `<name>@latest` line per tool. The Containerfile gains Layer 3-5 (`mise install` reading that list). Tools land in `~/.local/share/mise` (the `dotfiles_mise` named-volume mountpoint) and copy-on-first-mount into the volume on `make up`, matching the cargo/rustup persistence pattern.

**Tech Stack:** Python 3.14 + `tomllib` (codegen), pytest (tests), Bash + zsh + Podman/BuildKit (Containerfile), chezmoi (runtime dotfiles), mise (tool installer).

## Global Constraints

- All three mise tools pinned to **`latest`** (design dialogue Q1 ŌåÆ A). The generator renders `<name>@latest`; bare `mise install <tool>` reads a `mise.toml` (not latest), so the `@latest` suffix is mandatory.
- `dependencies/packages.toml` is the **only hand-edited** package manifest (invariant I-FS3); every `dependencies/layer_*/` file is generated by `make gen-deps` and must never be hand-edited.
- Each `[[tool]]` entry requires `name` (str), `manager` (one of the allowed enum), `layer` (int Ōēź 0), `has_configs` (bool); `description` (str) is optional.
- Containerfile `RUN` shells are `zsh -c`; word-splitting uses `${=pkgs}` (zsh split operator), **not** `$pkgs` (zsh does not word-split bare params ŌĆö the `neovim-git` AUR regression `a115677` was this exact bug).
- Every `toolchain`-stage `RUN` sources `/tmp/build-home/.zshenv` first (each RUN is a fresh shell; this materializes `MISE_DATA_DIR` and puts `~/.local/bin` on PATH so `mise` resolves).
- BuildKit cache mounts use `uid=${HOST_UID},gid=${HOST_GID}` (Podman Ōēź 4.0 requirement); mounts are not written to image layers.
- Commits: one commit per Phase. Commit message references the issue path `docs/issues/2026-07-01-mise-managed-languages.md`.

        {"name": "deno", "manager": "mise", "layer": 3, "has_configs": False,

    assert "go@latest" in out
    assert "deno@latest  # Deno runtime" in out

def test_write_txt_files_creates_layer_3_mise_txt(tmp_path, monkeypatch) -> None:
    monkeypatch.setattr(main, "DEPS_DIR", tmp_path)
    monkeypatch.setattr(main, "EXPECTED_EMPTY_FILES", ((3, "mise"),))
    by_layer = {
        3: [{"name": "go", "manager": "mise", "layer": 3,
             "has_configs": False, "description": "Go"}],
    }
    n = main.write_txt_files(by_layer)
    out = tmp_path / "layer_3" / "mise.txt"
    assert n == 1
    assert out.is_file()
    body = out.read_text()
    assert "manager: mise" in body
    assert "go@latest" in body

    tmp_path, monkeypatch
) -> None:

    assert out.is_file()

    assert "packages: 0" in body

Insert the following block **between** the end of Layer 3-4 and the Stage 4 divider:

```dockerfile

# Layer 3-5: install mise-managed languages from the generated list.
#
# `dependencies/` is outside the container/ build context, so it is
# exposed via the named BuildKit context `deps` (see Makefile) and read
# with `COPY --from=deps`. Comments and blank lines are stripped before
# `mise install`. The list contains only `manager = "mise"` entries,
# rendered as `<name>@latest` (bare `mise install <tool>` would read a
# mise.toml instead of latest, so the @latest suffix is required).
# `mise` is on PATH via ~/.local/bin (set in .zshenv); MISE_DATA_DIR is
# sourced from /tmp/build-home/.zshenv. Tools install into
# ~/.local/share/mise (the dotfiles_mise named-volume mountpoint), so
# they copy-on-first-mount into the volume on the first `make up`
# (spec 21 acceptance #8 analog). A BuildKit cache mount on ~/.cache/mise
# keeps mise's download cache across rebuilds without bloating image
# layers (mirrors the cargo cache mounts in 3-4). `${=pkgs}` (zsh split)
# restores per-package args.
COPY --from=deps layer_3/mise.txt /tmp/mise_tools.txt
RUN --mount=type=cache,target=/home/${USERNAME}/.cache/mise,uid=${HOST_UID},gid=${HOST_GID} \
    zsh -c 'set -eo pipefail; \
      source /tmp/build-home/.zshenv; \
      pkgs=$(sed "s/#.*//" /tmp/mise_tools.txt | xargs); \
      if [ -n "$pkgs" ]; then \
        mise install ${=pkgs}; \
        mise use -g ${=pkgs}; \
      else \
