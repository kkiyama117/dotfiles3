# `paru` (AUR) Install Layer — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dedicated `aur` stage (Layer 4) to the container build that bootstraps `paru` from the AUR via `makepkg` and installs a generated AUR package list from `dependencies/layer_4/paru.txt`, fed from `dependencies/packages.toml`; renumber the `runtime` stage to Layer 5; resolve spec 21 Q1 and spec 20 Q2.

**Architecture:** 5-stage Containerfile (`base → build-prepass → toolchain → aur → runtime`); the new `aur` stage is `FROM toolchain AS aur`, runs as non-root `${USERNAME}`, bootstraps `paru` via `makepkg -si` against an AUR clone, then `paru -S --noconfirm --needed` the generated list with BuildKit `--mount=type=cache` on `~/.cache/paru` and `/var/cache/pacman/pkg`. `runtime` becomes `FROM aur AS runtime` (Layer 5) and the bind dir is renamed `layer_4_files` → `layer_5_files`.

**Tech Stack:** Podman ≥ 4.0 / BuildKit, Manjaro base image, `paru` / `makepkg`, AUR, Python 3.11+ tomllib, chezmoi.

**Spec:** [`docs/specifications/implementations/2026-06-30-paru-aur-layer-design.md`](../specifications/implementations/2026-06-30-paru-aur-layer-design.md)
**Parent issue:** [`docs/issues/2026-06-30-paru-aur-layer.md`](../issues/2026-06-30-paru-aur-layer.md)

## Global Constraints

- **Secret-free image** (spec 13 / spec 20 I4): the `aur` stage installs only system packages and never consults Bitwarden. No change to the build-prepass / runtime apply contract.
- **All packages from `packages.toml`** (spec 20 I5/I8): every AUR package, including `paru` itself, is declared in `dependencies/packages.toml`; `dependencies/layer_4/paru.txt` is generated by `make gen-deps`, never hand-edited.
- **Non-root build** (spec 20 I7): `makepkg` and `paru` run as `${USERNAME}`; root escalation for `pacman` happens only via the Layer 1-4 NOPASSWD sudoers entry.
- **Podman ≥ 4.0** required for `uid=` / `gid=` on `--mount=type=cache` (already required by the toolchain stage).
- **`USERNAME` defined in `.env`** at repo root (existing). `make build` / `make up` abort when unset.
- **Numeric layer order == build-chain order** (spec 21 convention): inserting `aur` at Layer 4 forces `runtime` to Layer 5 and renames `container/bind/layer_4_files/` → `layer_5_files/` in the same change.
- **Initial `paru.txt` = `{ paru, neovim-git }`** — `paru` (the bootstrap tool, I5) plus one concrete AUR package (`neovim-git`, a -git build of neovim from upstream master) to exercise the full AUR clone/build path end-to-end. Additional AUR packages are follow-up commits.

## Task 4: Add the `aur` stage and renumber `runtime` to Layer 5 in the Containerfile

**Files:**
- Modify: `container/Containerfile`

**Interfaces:**
- Consumes: `dependencies/layer_4/paru.txt` (Task 2) via `--from=deps`; `USER ${USERNAME}` inherited from `toolchain`; NOPASSWD sudoers from Layer 1-4; `container/bind/layer_5_files/entrypoint.sh` (Task 3)
- Produces: a 5-stage build with `paru` on PATH in the final image; `runtime` is `FROM aur AS runtime` (Layer 5).

- [ ] **Step 4.1: Insert the `aur` stage between `toolchain` and `runtime`**

In `container/Containerfile`, find the Stage 4 `runtime` block beginning with:

```dockerfile
# ---------------------------------------------------------------------------
# Stage 4: runtime (final)
```

Insert **immediately before** it:

```dockerfile

# ---------------------------------------------------------------------------
# Stage 4: aur
#
# Bootstrap paru from the AUR (paru is itself an AUR package and cannot
# be installed via pacman), then install the Layer 4 AUR package set from
# dependencies/layer_4/paru.txt (the single source of truth per spec 20
# I5/I8; generated by `make gen-deps`). Runs as non-root ${USERNAME};
# root escalation for pacman happens only via the Layer 1-4 NOPASSWD
# sudoers (spec 20 I7). BuildKit cache mounts keep the AUR clone/build
# cache and the pacman package cache across rebuilds without bloating
# image layers (resolves spec 21 Q1 / spec 20 Q2).
# ---------------------------------------------------------------------------
FROM toolchain AS aur

# Layer 4-1: bootstrap paru via makepkg (non-root).
#
# paru is an AUR package, so install it the AUR way: clone the PKGBUILD,
# run makepkg -si (which installs paru + its repo deps via sudo pacman).
# `sudo pacman -Sy` refreshes the sync DB so makepkg's dependency
# resolution succeeds (no `-u`: Layer 1-2 already ran `pacman -Syu`).
# /tmp/paru-build is removed at the end so the clone never enters the
# image layer (I-AUR4).
RUN --mount=type=cache,target=/var/cache/pacman/pkg \
    --mount=type=cache,target=/home/${USERNAME}/.cache/paru,uid=${HOST_UID},gid=${HOST_GID} \
    set -e; \
    sudo pacman -Sy --noconfirm; \
    git clone https://aur.archlinux.org/paru.git /tmp/paru-build; \
    cd /tmp/paru-build && makepkg -si --noconfirm --needed; \
    rm -rf /tmp/paru-build

# Layer 4-2: install the AUR package set from the generated list.
#
# `dependencies/` is outside the container/ build context, so it is
# exposed via the named BuildKit context `deps` (see Makefile) and read
# with `COPY --from=deps`. Comments and blank lines are stripped before
# `paru -S`. The list includes the `paru` line itself; `--needed` makes
# it a no-op once paru is bootstrapped, exercising the install-list path
# without rebuilding paru.
COPY --from=deps layer_4/paru.txt /tmp/paru_deps.txt
RUN --mount=type=cache,target=/var/cache/pacman/pkg \
    --mount=type=cache,target=/home/${USERNAME}/.cache/paru,uid=${HOST_UID},gid=${HOST_GID} \
    set -e; \
    pkgs=$(sed 's/#.*//' /tmp/paru_deps.txt | xargs); \
    if [ -n "$pkgs" ]; then \
      paru -S --noconfirm --needed $pkgs; \
    else \
      echo "aur: paru install list is empty -- skipping"; \
    fi
```
